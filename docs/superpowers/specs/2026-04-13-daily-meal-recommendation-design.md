# 每日饮食推荐功能设计

## 概述

为现有蛋白质成本计算器新增"一键生成每日饮食方案"功能。用户设定每日蛋白质目标后，系统自动推荐早餐、午餐、晚餐和加餐的食材组合，兼顾性价比和饮食多样性。

移动端优先，作为独立页面，不修改现有代码。

## 需求

- 一键生成全天四餐（早/午/晚/加餐）食材组合
- 用户可设定固定蛋白质目标（如 120g/天）
- 四餐按固定比例分配蛋白质：早 25%、午 35%、晚 30%、加餐 10%
- 基于现有食材数据库（22 种）用贪心算法组合
- 连续多天推荐时避免食材重复（7 天去重）
- 每餐可单独"换一套"，也可全部重新生成

## 页面布局

移动端单列布局，从上到下：

1. **顶栏**：标题"每日饮食推荐" + 当日日期 + 设置入口
2. **目标卡片**：今日蛋白质目标、进度条（已选/目标）、预估总花费
3. **四张餐次卡片**：早餐/午餐/晚餐/加餐
   - 左侧彩色边条区分餐次（早餐黄/午餐红/晚餐蓝/加餐紫）
   - 食材列表（名称 + 食用量）
   - 小计蛋白质 + 花费
   - "换一套"按钮
4. **底部**："全部重新生成"按钮

## 推荐算法

### 核心流程

```
用户点击"生成"
  → 读取蛋白质目标
  → 按比例拆分到四餐
  → 对每餐独立调用 selectMeal(targetProtein)
  → 渲染结果
```

### selectMeal(targetProtein)

1. **计算权重**：
   - 基础权重 = `1 / cost_per_500g`（性价比越高越容易被选中）
   - 重复衰减 = 近 7 天已用次数 × 0.3 的衰减因子
   - 最终权重 = 基础权重 × 重复衰减

2. **贪心填充**：
   - 按权重随机选 1 个主蛋白源（提供 60-80% 目标蛋白质）
   - 按权重随机选 1-2 个辅助食材补足剩余量
   - 每个食材按常见食用量取整

3. **结果校验**：
   - 蛋白质总量在目标 ±15% 内
   - 超出范围则重试（最多 10 次）

4. **记录使用**：写入 localStorage，记录日期 + 食材

### 常见食用量映射

| 食材类型 | 单位 |
|---------|------|
| 鸡蛋 | 个（约 50g）|
| 肉类 | 100g / 150g |
| 牛奶/豆浆 | 250ml / 300ml |
| 豆腐 | 100g / 150g |
| 坚果 | 30g |
| 蛋白粉 | 一勺（约 30g）|

## 数据模型

### 新增 localStorage 数据

```javascript
// 用户设置
recommendSettings: {
  proteinTarget: 120,
  ratios: [0.25, 0.35, 0.30, 0.10],
}

// 推荐历史（最多保留 7 天）
recommendHistory: [
  {
    date: "2026-04-13",
    meals: [
      {
        slot: "breakfast",
        items: [
          { name: "鸡蛋", amount: "×2", grams: 100, protein: 13.3, cost: 1.2 }
        ]
      },
      ...
    ]
  },
]

// 当日推荐结果
todayRecommend: { ... }
```

### 与现有数据的关系

- 直接复用 `data.js` 中的 `defaultFoods`（只读）
- 不改动现有数据结构
- 推荐算法为纯函数：输入（食材列表 + 设置 + 历史）→ 输出（推荐结果）

## 文件组织

| 文件 | 说明 |
|------|------|
| `recommend.html` | 新建，移动端推荐页面 |
| `recommend.js` | 新建，算法逻辑 + 渲染 |
| `data.js` | 不改动，直接引用 |

与现有 `index.html`、`mobile.html` 完全独立。可在现有页面中加一个链接入口到 `recommend.html`。

## 设置页面

点击顶栏设置按钮进入：
- 蛋白质目标：数字输入框，默认 120g
- 四个餐次比例：slider 展示，拖动一个其余自动调整，总和始终 100%
- 保存后立即生效

## 交互细节

- 单张卡片"换一套"：只重新生成该餐
- "全部重新生成"：四餐全部重做
- 点击后直接替换显示，无确认弹窗
- 推荐结果存入历史，用于后续去重
